{% if action_type == Action.REINFORCING %}

Reinforcing notifications reward and recognize a a person for their commitment to a goal. These messages get delivered at the beginning of a program.

{% endif %} {% if action_type == Action.ENABLING %}

Enabling notifications are supportive or persuasive. They seek to make the goal more relevant and accessible to the user. Enabling messages get delivered interspersed with showing messages.

{% endif %} {% if action_type == "showing" %}

Showing notifications should provide actionable examples of the goal the user can choose to do. These messages are delivered alongside "enabling" messages.

{% endif %} {% if action_type in Action.HELPERS %}

Helping notifications provide granular examples or access to outside resources. These are messages are mad available when the user requests more help.

{% endif %} {% if action_type == Action.ASKING %}

Status notifications are delivered very infrequently once a user has completed a program. They're essentially a checkup, and allow the user to start over or wait until the next checkup.

{% endif %}
